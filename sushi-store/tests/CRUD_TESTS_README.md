# CRUD Тесты для Sushi Store

## Обзор

Этот набор тестов покрывает все CRUD (Create, Read, Update, Delete) операции для всех сущностей в приложении Sushi Store. Тесты проверяют:

- ✅ Создание записей
- ✅ Чтение записей  
- ✅ Обновление записей
- ✅ Удаление записей
- ✅ Валидацию данных
- ✅ Авторизацию и безопасность

## Структура тестов

### 1. CRUD тесты для продуктов (`crud-products.test.js`)
**Покрывает:** Управление товарами в ресторане

**Тестируемые операции:**
- Создание продукта с валидными/невалидными данными
- Получение списка продуктов и отдельного продукта
- Обновление всех полей продукта (название, описание, цена, изображение, категория, доступность)
- Удаление продукта
- Проверка авторизации

**Поля продукта:**
- `name` - название продукта
- `description` - описание
- `price` - цена
- `image` - изображение
- `category_id` - ID категории
- `available` - доступность

### 2. CRUD тесты для категорий (`crud-categories.test.js`)
**Покрывает:** Управление категориями товаров

**Тестируемые операции:**
- Создание категории с валидными/невалидными данными
- Получение списка категорий и отдельной категории
- Обновление всех полей категории (название, описание, изображение)
- Удаление категории (с проверкой зависимостей)
- Проверка авторизации

**Поля категории:**
- `name` - название категории
- `description` - описание
- `image` - изображение
- `is_active` - активность

### 3. CRUD тесты для заказов (`crud-orders.test.js`)
**Покрывает:** Управление заказами клиентов

**Тестируемые операции:**
- Создание заказа с валидными/невалидными данными
- Получение списка заказов и отдельного заказа
- Обновление всех полей заказа (клиент, статус, оплата, доставка, заметки)
- Удаление заказа
- Проверка авторизации

**Поля заказа:**
- `customer_name` - имя клиента
- `customer_phone` - телефон клиента
- `customer_address` - адрес клиента
- `total_amount` - общая сумма
- `status` - статус заказа
- `delivery_time` - время доставки
- `persons` - количество персон
- `extras_selection` - выбор допов
- `notes` - заметки
- `paid` - статус оплаты

### 4. CRUD тесты для новостей (`crud-news.test.js`)
**Покрывает:** Управление новостями и статьями

**Тестируемые операции:**

**Поля новости:**
- `title` - заголовок
- `content` - содержание

- `published` - статус публикации

### 5. CRUD тесты для отзывов (`crud-reviews.test.js`)
**Покрывает:** Управление отзывами клиентов

**Тестируемые операции:**
- Создание отзыва с валидными/невалидными данными
- Получение списка отзывов и отдельного отзыва

**Покрывает:** Управление SEO настройками страниц

**Тестируемые операции:**

### 7. CRUD тесты для блоков категорий (`crud-category-blocks.test.js`)
**Покрывает:** Управление блоками категорий на главной странице

**Тестируемые операции:**
- Создание блока категории с валидными/невалидными данными
- Получение списка блоков и отдельного блока
- Обновление всех полей блока (заголовок, описание, изображение, порядок, активность)
- Удаление блока

**Поля блока категории:**

- `description` - описание блока
- `image` - изображение блока
- `category_id` - ID категории
- `order_index` - порядок отображения
- `is_active` - активность блока

## Запуск тестов

### Запуск всех CRUD тестов:
```bash
node test-runner.js crud
```

### Запуск отдельных тестов:
```bash
# Тесты продуктов
node test-runner.js crud-products

# Тесты категорий
node test-runner.js crud-categories

# Тесты заказов
node test-runner.js crud-orders

# Тесты новостей
node test-runner.js crud-news

# Тесты отзывов
node test-runner.js crud-reviews

# Тесты SEO
node test-runner.js crud-seo

# Тесты блоков категорий
node test-runner.js crud-category-blocks
```

### Запуск через Jest напрямую:
```bash
# Все CRUD тесты
npm test -- tests/crud-*.test.js

# Отдельный тест
npm test -- tests/crud-products.test.js
```

## Требования

- ✅ Сервер должен быть запущен на `http://localhost:3000`
- ✅ База данных PostgreSQL должна быть настроена
- ✅ Админ пользователь должен существовать (`+7 (999) 123-45-67` / `admin123`)
- ✅ Все API endpoints должны быть доступны

## Особенности тестирования

### Авторизация
Все тесты автоматически:
1. Логинятся как админ
2. Получают JWT токен
3. Используют токен для всех запросов
4. Проверяют, что неавторизованные запросы отклоняются

### Валидация данных
Тесты проверяют:
- ✅ Обязательные поля
- ✅ Формат данных (email, телефон, рейтинг)
- ✅ Ограничения длины
- ✅ Числовые ограничения (цена, рейтинг)
- ✅ Перечисления (статусы, роли)

### Очистка данных
Тесты автоматически:
- ✅ Создают тестовые данные
- ✅ Удаляют тестовые данные после завершения
- ✅ Не влияют на существующие данные

### Обработка ошибок
Тесты проверяют:
- ✅ 400 Bad Request для невалидных данных
- ✅ 401 Unauthorized для неавторизованных запросов
- ✅ 404 Not Found для несуществующих ресурсов
- ✅ 500 Internal Server Error для серверных ошибок

## Отладка

Если тесты не проходят:

1. **Проверьте сервер:** `curl http://localhost:3000/api/products`
2. **Проверьте авторизацию:** `curl -X POST http://localhost:3000/api/login -d '{"phone":"+7 (999) 123-45-67","password":"admin123"}'`
3. **Проверьте базу данных:** подключитесь к PostgreSQL и проверьте таблицы
4. **Проверьте логи сервера:** смотрите консоль сервера на ошибки

## Расширение тестов

Для добавления новых тестов:

1. Создайте новый файл `tests/crud-[entity].test.js`
2. Следуйте структуре существующих тестов
3. Добавьте новый тип теста в `test-runner.js`
4. Обновите документацию
